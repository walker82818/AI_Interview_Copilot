# AI Interview Copilot - 项目亮点总结

---

## 🎯 亮点一：全链路 AI 面试流程 — 从简历到评分的一站式闭环

不同于市面上仅提供"AI 对话"的面试工具，本项目的核心亮点是**覆盖面试全生命周期的完整闭环**：

```
简历上传 → AI 解析简历 → JD 技能分析 → 简历×JD 匹配度计算
       → RAG 增强面试追问 → 流式对话交互 → 多维度评分报告 → 雷达图可视化
```

**每一个环节都由 AI 驱动，而非简单的规则匹配**：
- **简历解析**：PyMuPDF 提取 PDF 文本后，调用 LLM 自动提取技能树、工作经历、教育背景，而非简单正则匹配
- **JD 分析**：LLM 分析岗位必备技能 vs 加分技能，自动生成技能权重
- **面试追问**：基于候选人简历 + JD 的向量检索结果，LLM 生成个性化追问，而非死板的题库
- **评分报告**：面试结束后 LLM 多维度评估（技术能力、沟通表达、问题解决等），生成雷达图 + 学习建议

---

## 🧠 亮点二：RAG（检索增强生成）驱动的智能面试追问

这是项目最核心的技术亮点。面试问题不是从题库中随机抽取，而是**基于候选人的真实简历和岗位需求动态生成**：

```
用户简历/简历分块 → Embedding API 向量化 → Milvus 向量存储
                         ↓
面试官提问时 → Milvus 向量检索 (COSINE) → Rerank 重排序 → LLM 生成追问
```

**技术栈组合**：
- **Embedding**：阿里百炼 Embedding API 进行文本向量化
- **向量数据库**：Milvus 2.5+（分布式、高性能，支持亿级向量检索）
- **Rerank**：重排序层（预留 BGE-Reranker 接入点）提升检索精度
- **Prompt 工程**：RAGService 构建上下文注入 Prompt，让 LLM 生成基于真实背景的追问

**区别于传统方案的优势**：
| 传统方案 | 本项目 RAG 方案 |
|---------|---------------|
| 固定题库，千篇一律 | 动态生成，千人千面 |
| 不感知候选人背景 | 基于简历内容精准追问 |
| 无法评估岗位匹配 | 结合 JD 要求深度挖掘 |

## ⚡ 亮点三：流式响应（SSE）+ 实时面试对话体验

面试对话采用 **Server-Sent Events (SSE) 流式传输**，而非传统的请求-响应模式：

- **后端**：FastAPI `StreamingResponse` + `text/event-stream`，LLM 生成的每个 token 实时推送
- **前端**：原生 `EventSource` / `fetch` 流式读取，逐字渲染 AI 面试官回复
- **用户体验**：候选人能实时看到 AI 面试官"正在打字"，大幅提升对话自然感

```
POST /api/interviews/{id}/messages/stream
  → 200 OK (text/event-stream)
  → data: {"type":"token","content":"请"}
  → data: {"type":"token","content":"问"}
  → data: {"type":"token","content":"..."}
  → data: {"type":"done"}
```

## 🔐 亮点四：生产级安全架构设计

项目在安全方面做了**远超 Demo 级别**的设计：

| 安全措施 | 实现方式 |
|---------|---------|
| **JWT 认证** | python-jose HS256 签名，access_token + refresh_token 双令牌 |
| **密码安全** | bcrypt 12 轮哈希，不可逆存储 |
| **令牌轮换** | 登出时令牌加入黑名单，防止重放攻击 |
| **SQL 注入防护** | SQLAlchemy ORM 参数化查询，彻底杜绝拼接 SQL |
| **请求限流** | 登录接口频率限制，防止暴力破解 |
| **CORS 配置** | 严格的跨域白名单，仅允许前端域名 |
| **环境隔离** | .env 配置管理，敏感信息不入库 |
| **输入验证** | Pydantic 严格 Schema 校验，拒绝非法输入 |

## 🏗️ 亮点五：分层架构设计 — V1 务实 + V2 前瞻

项目采用**清晰的分层架构**，体现了良好的软件工程素养：

```
┌─────────────────────────────────────┐
│           API Layer (路由层)          │  ← FastAPI Router，仅做参数校验和响应
├─────────────────────────────────────┤
│         Service Layer (服务层)        │  ← 核心业务逻辑，可复用、可测试
├─────────────────────────────────────┤
│          Core Layer (核心层)          │  ← 数据库连接、安全中间件、配置
├─────────────────────────────────────┤
│         Agent Layer (智能体层)        │  ← V2 预留，多步推理、状态机
└─────────────────────────────────────┘
```

**V1 vs V2 演进策略**：
- **V1（已实现）**：`services/` 直接编排 LLM + RAG 调用，快速验证核心链路
- **V2（预留架构）**：`agents/` 层已规划 4 个 Agent（ResumeAgent / JDAgent / InterviewAgent / EvaluationAgent），支持多步推理、面试状态机、难度自适应等高级能力

这种设计体现了**先跑通核心流程，再逐步智能化**的务实工程思路。

## 🎨 亮点六：现代化前端技术栈与交互体验

| 技术选型 | 亮点说明 |
|---------|---------|
| **Next.js 16 App Router** | 最新版 Next.js，React Server Components 架构 |
| **Zustand + persist** | 轻量状态管理，localStorage 持久化，比 Redux 简洁 80% |
| **Tailwind CSS 4** | 原子化 CSS，开发效率极高 |
| **TypeScript 全栈** | 前后端统一类型体系，`types/` 目录定义共享接口 |
| **自定义 Hooks** | `useAuth` 等自定义 Hook 封装认证逻辑，组件解耦 |

**页面设计覆盖完整用户旅程**：
- `login/` & `register/` — 认证入口
- `dashboard/` — 数据仪表板（简历数、JD 数、最近面试）
- `resume/` — 简历上传与管理
- `jd/` — JD 上传与技能分析
- `interview/` — 核心面试对话页
- `report/` — 评分报告与雷达图

## 📊 亮点七：多维度 AI 评分与可视化报告

面试结束后，系统自动生成**专业的多维度评分报告**：

- **评分维度**：技术能力、沟通表达、问题解决能力、学习能力、团队协作等
- **雷达图可视化**：直观展示候选人在各维度的表现
- **学习建议**：LLM 根据弱项生成针对性的改进建议
- **面试回溯**：完整对话记录 + 逐题点评

```
面试结束 → Evaluation Agent 分析对话 → 多维度评分
       → 雷达图生成 → 学习建议 → 完整报告
```

## 🐳 亮点八：Docker Compose 一键部署 + 多服务编排

```
docker-compose up -d
```

一条命令启动完整基础设施：

| 服务 | 角色 |
|------|------|
| PostgreSQL 15 | 关系数据库 |
| Milvus Standalone | 向量数据库 |
| etcd | Milvus 元数据存储 |
| MinIO | Milvus 对象存储 |

前后端可独立开发部署，Docker 化基础设施实现**环境一致性**。

## 🔧 亮点九：完善的工程化实践

- **日志系统**：loguru 结构化日志，支持日志轮转和分级
- **异步架构**：FastAPI async/await + SQLAlchemy async + httpx async，全链路非阻塞
- **错误处理**：统一的异常处理中间件，规范化错误响应格式
- **Mock 降级**：LLMService 内置 mock 模式，无 API Key 也能跑通流程
- **数据库迁移**：Alembic 管理 schema 变更，支持版本回滚
- **测试框架**：pytest + pytest-asyncio，支持异步测试
